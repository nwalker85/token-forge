# token-forge

Transform design token YAML files into platform-specific output formats. One source of truth, every platform covered.

```
YAML theme file ──→ token-forge ──→ CSS custom properties
                                ──→ Tailwind config
                                ──→ Dart classes (via Loki)
```

## Why

Design tokens get duplicated across every project. You change a color in one place and forget three others. token-forge fixes this by generating platform-specific token files from a single YAML source.

The YAML schema is borrowed from [Loki](https://github.com/ravenhelm/loki)'s token system — a battle-tested structure covering colors, spacing, radii, typography, glassmorphism, and type scales. Loki already parses this YAML natively in Dart. token-forge extends that same schema to CSS and Tailwind.

## Install

```bash
# Global CLI
npm install -g @nwalker/token-forge

# Project dependency
npm install -D @nwalker/token-forge
```

## Quick Start

Given a token file `tokens/dark.yaml`:

```yaml
name: my-theme
version: 1.0.0

colors:
  background: "#0E1014"
  surface: "#12141A"
  primary: "#7FA3C8"
  text:
    primary: "#E6EAF2"
    secondary: "#B6BDCB"
    muted: "#8690A5"
  glass:
    background: "rgba(18,20,26,0.80)"
    border: "rgba(255,255,255,0.10)"
    highlight: "rgba(255,255,255,0.05)"
  status:
    success: "#5BB890"
    warning: "#C4A45E"
    error: "#C47272"

spacing:
  xs: 4
  sm: 8
  md: 12
  lg: 16
  xl: 24
  xxl: 32

radii:
  sm: 4
  md: 6
  lg: 8
  xl: 12
  full: 999

typography:
  family: Inter
  display_family: Plus Jakarta Sans
  mono_family: JetBrains Mono
  scales:
    display_large: { size: 32, weight: 700, spacing: -0.5 }
    body_medium:   { size: 14, weight: 400 }

glass:
  blur_sigma: 12
  border_width: 1
```

Generate CSS:

```bash
token-forge build -i tokens/dark.yaml -o src/tokens.css
```

Output:

## CLI Reference

```
token-forge build [options]

Options:
  -i, --input <file>        Input YAML token file (required)
  -o, --output <file>       Output file (default: stdout)
  -f, --format <format>     Output format: css, tailwind (default: css)
  --prefix <prefix>         CSS variable prefix (default: --)
  --selector <selector>     CSS selector scope (default: :root)
  --color-format <format>   Color format: hsl, hex (default: hsl)
```

### Examples

```bash
# CSS with HSL colors (default, Tailwind-compatible)
token-forge build -i theme.yaml -o tokens.css

# CSS with raw hex values (for non-Tailwind projects)
token-forge build -i theme.yaml -o tokens.css --color-format hex

# Tailwind theme extension
token-forge build -i theme.yaml -f tailwind -o tailwind.tokens.js

# Scoped to a CSS class instead of :root
token-forge build -i theme.yaml --selector ".theme-dark" -o dark.css

# Custom variable prefix
token-forge build -i theme.yaml --prefix "--rh-" -o tokens.css

# Pipe to stdout
token-forge build -i theme.yaml | pbcopy
```

## Output Formats

### CSS (`-f css`)

Generates CSS custom properties under `:root` (or a custom selector). Colors default to HSL space-separated values for Tailwind compatibility. Glass/rgba colors preserve their alpha channel.

Use in HTML:
```html
<link rel="stylesheet" href="tokens.css">
```

Use in CSS:
```css
.card {
  background: hsl(var(--surface));
  color: hsl(var(--text-primary));
  border: 1px solid hsl(var(--glass-border));
  border-radius: var(--radius-lg);
  padding: var(--spacing-lg);
  backdrop-filter: blur(var(--glass-blur));
}
```

### Tailwind (`-f tailwind`)

Generates a Tailwind theme extension that references the CSS custom properties. Designed to work alongside the CSS output — the CSS file defines the values, the Tailwind config maps utility classes to them.

```bash
# Generate both
token-forge build -i theme.yaml -o src/tokens.css
token-forge build -i theme.yaml -f tailwind -o tailwind.tokens.js
```

Import in `tailwind.config.ts`:
```typescript
import tokens from './tailwind.tokens.js';

export default {
  ...tokens,
  content: ['./src/**/*.{html,tsx}'],
};
```

Then use in markup:
```html
<div class="bg-surface text-text-primary rounded-lg p-lg">
  <h1 class="font-display text-primary">Hello</h1>
</div>
```

## Programmatic API

token-forge also exports its parser and formatters for use in build scripts:

```typescript
import { parseTokens, formatCss, formatTailwind } from '@nwalker/token-forge';
import { readFileSync } from 'fs';

const yaml = readFileSync('theme.yaml', 'utf8');
const tokens = parseTokens(yaml);

const css = formatCss(tokens, { format: 'hsl', selector: ':root' });
const tailwind = formatTailwind(tokens);
```

## Token Schema

The YAML schema covers everything you need for a modern UI:

| Section | Keys | Description |
|---------|------|-------------|
| `colors` | background, surface, primary, text.*, glass.*, status.* | Full color palette with semantic grouping |
| `spacing` | xs, sm, md, lg, xl, xxl, xxxl | Spacing scale in px |
| `radii` | sm, md, lg, xl, full | Border radius scale in px |
| `typography` | family, display_family, mono_family, scales.* | Font families and type scale definitions |
| `glass` | blur_sigma, border_width | Glassmorphism effect parameters |
| `entity_types` | (any) | Domain-specific entity colors/icons (optional) |
| `components` | (any) | Component-level token overrides (optional) |

### Color Groups

```yaml
colors:
  # Core surfaces
  background: "#0E1014"       # Page/app background
  surface: "#12141A"          # Card/panel background
  surface_variant: "#171A21"  # Elevated surface

  # Edges
  edge: "#2A2A2A"             # Default borders
  edge_light: "#3A3A3A"       # Subtle borders

  # Brand
  primary: "#7FA3C8"          # Primary accent
  primary_light: "#96B6D5"    # Light variant
  primary_glow: "#89C7DE"     # Glow/hover state
  tertiary: "#4C1D95"         # Secondary accent
  tertiary_light: "#7C3AED"   # Light variant

  # Text hierarchy
  text:
    primary: "#E6EAF2"        # Body text
    secondary: "#B6BDCB"      # Supporting text
    muted: "#8690A5"          # Disabled/hint text

  # Glassmorphism
  glass:
    background: "rgba(18,20,26,0.80)"
    border: "rgba(255,255,255,0.10)"
    highlight: "rgba(255,255,255,0.05)"

  # Semantic
  status:
    success: "#5BB890"
    warning: "#C4A45E"
    error: "#C47272"
    info: "#7FA3C8"
```

## Multi-Platform Workflow

token-forge is one piece of a larger token pipeline:

```
Figma ──→ figma_token_extractor.py ──→ theme.yaml ──→ token-forge ──→ CSS
                                                                    ──→ Tailwind
                                                   ──→ LokiTokenParser ──→ Flutter
```

The same YAML file feeds both web (via token-forge) and Flutter (via Loki's native Dart parser). No manual sync. No drift.

### Build Script Example

```json
{
  "scripts": {
    "tokens": "token-forge build -i tokens/theme.yaml -o src/tokens.css",
    "tokens:tailwind": "token-forge build -i tokens/theme.yaml -f tailwind -o tailwind.tokens.js",
    "build": "npm run tokens && vite build"
  }
}
```
